# CSS 스터디: overflow와 absolute 포지셔닝 — 드롭다운 잘림 현상

> **스터디 목적**: 부모의 `overflow`가 자식의 `position: absolute`에 미치는 영향을 이해하고, 드롭다운·팝오버가 잘리는 문제를 해결할 수 있다.

---

## 1. 현상 요약

- **상황**: 필터 패널(`PaymentRecordFilters`) 안에 회원 다중 선택 드롭다운(`MemberMultiSelectDropdown`)이 있음. 드롭다운을 열면 리스트가 부모 영역에 잘려서 보이지 않음.
- **원인**: 필터 패널 루트에 `overflow: hidden`이 적용되어 있어, 내부의 `position: absolute` 드롭다운이 부모의 박스 밖으로 나가지 못하고 잘림.

---

## 2. CSS 개념 정리

### 2.1 `overflow`가 하는 일

- `overflow`는 **해당 요소가 만든 블록 서식 맥락(block formatting context) 안에서**, 내용이 **요소의 콘텐츠 박스(content box)를 넘어갈 때** 어떻게 보일지/잘릴지/스크롤할지를 결정한다.
- `overflow: hidden` / `overflow: auto` / `overflow: scroll`인 요소는 **스크롤 포터(scrollport)**가 되며, **그 밖으로 나가는 내용은 잘린다.**

### 2.2 `position: absolute`와 “잘림”의 관계

- `position: absolute`인 요소는 **가장 가까운 `position`이 `relative`/`absolute`/`fixed`/`sticky`인 조상**, 또는 **스크롤 포터(overflow가 hidden/auto/scroll인 조상)**를 기준으로 위치할 수 있다.
- 중요한 점: **스크롤 포터(overflow로 잘라내는 조상) 안에 있으면**, absolute 요소가 그 조상의 **시각적 경계 밖**으로 나가더라도 **화면에는 그 경계까지만 보이고, 나머지는 잘린다.**

즉, “드롭다운이 부모 div에 짤린다”는 현상은:

- 드롭다운: `position: absolute`
- 그 조상 중 하나: `overflow: hidden` (또는 auto/scroll)
- → 그 조상이 “가위” 역할을 해서, 밖으로 삐져나온 부분을 잘라냄.

---

## 3. 실제 코드에서의 원인

### 3.1 부모 쪽 (잘리게 만든 요소)

```tsx
// PaymentRecordFilters.tsx — 수정 전
<div className="mb-4 border rounded-lg bg-gray-50 overflow-hidden">
  {/* ... 필터 버튼, 필터 내용 ... */}
  <MemberMultiSelectDropdown ... />
</div>
```

- 루트 div에 `overflow-hidden`(Tailwind → `overflow: hidden`)이 있음.
- 이 div가 **스크롤 포터**가 되어, 그 안의 모든 내용(absolute 드롭다운 포함)이 이 박스 밖으로 나가면 **시각적으로 잘림**.

### 3.2 자식 쪽 (잘린 요소)

```tsx
// MemberMultiSelectDropdown.tsx
<div ref={dropdownRef} className="relative">
  <button>...</button>
  {isOpen && (
    <div className="absolute z-10 mt-1 w-full ...">{/* 드롭다운 리스트 */}</div>
  )}
</div>
```

- 드롭다운 패널이 `absolute`로 버튼 아래에 배치됨.
- 이 `absolute` 요소는 `overflow: hidden`인 조상(PaymentRecordFilters 루트) **안**에 있기 때문에, 조상 박스 아래로 삐져나온 부분이 잘림.

---

## 4. 해결 방법 (우리 프로젝트에서 적용한 방식)

**조상에서 잘라내지 않도록** `overflow-hidden`을 제거하고, 대신 **시각적 형태(둥근 모서리)**는 다른 클래스로 유지했다.

1. **루트에서 `overflow-hidden` 제거**
   - 드롭다운이 부모 박스에 의해 잘리지 않도록 함.

2. **라운드 처리 유지**
   - 접었을 때: 버튼만 보이므로 버튼에 `rounded-t-lg` + `rounded-b-lg`.
   - 펼쳤을 때: 버튼은 `rounded-t-lg`, 필터 내용 영역에 `rounded-b-lg` + `border-t`로 하단 모서리와 구분선 유지.

```tsx
// 수정 후
<div className="mb-4 border rounded-lg bg-gray-50">
  <button className={`... rounded-t-lg ${!isOpen ? 'rounded-b-lg' : ''}`}>
    ...
  </button>
  {isOpen && (
    <div className="... border-t border-gray-200 rounded-b-lg">...</div>
  )}
</div>
```

이렇게 하면 “overflow로 잘림” 없이 드롭다운이 정상적으로 보이고, 필터 패널 디자인도 유지된다.

---

## 5. 스터디용 대안 해결책

같은 현상을 해결하는 다른 방법들도 알아두면 좋다.

### 5.1 React Portal 사용

- 드롭다운 내용을 **DOM 트리에서는 부모 밖**(예: `document.body`)에 렌더링.
- 위치는 트리거 버튼의 `getBoundingClientRect()` 등으로 계산해서 `position: fixed`로 배치.
- **장점**: 상위에 `overflow: hidden`이 있어도 잘리지 않음.
- **단점**: 포지션 계산·리사이즈/스크롤 시 위치 갱신 등 로직이 필요함.

### 5.2 overflow를 가진 조상만 제거/변경

- “잘림”을 만드는 건 **overflow가 hidden/auto/scroll인 가장 가까운 조상**이므로, 그 조상만 바꿔도 됨.
- 예: 필터 패널에는 overflow를 두지 않고, **필터 패널과 형제 레벨**에만 overflow를 쓰는 구조로 레이아웃을 나누는 방법.

### 5.3 overflow: visible 유지가 불가능한 경우

- 디자인상 반드시 `overflow: hidden`이 필요한 영역이 있다면, **드롭다운/팝오버는 그 영역 밖에서 렌더링**하는 쪽(Portal 등)이 안전하다.

---

## 6. 정리

| 구분          | 내용                                                                                          |
| ------------- | --------------------------------------------------------------------------------------------- |
| **원인**      | 부모에 `overflow: hidden` → 그 안의 `position: absolute` 자식이 부모 밖으로 나간 부분이 잘림. |
| **해결**      | 해당 부모에서 `overflow-hidden` 제거하고, 둥근 모서리 등은 `rounded-*` 클래스로 유지.         |
| **핵심 개념** | `overflow`(hidden/auto/scroll)가 있는 요소는 스크롤 포터가 되며, 그 경계 밖의 내용은 잘린다.  |

나중에 “팝오버/드롭다운이 어디선가 잘린다”면, **잘린 요소부터 위로 올라가면서 `overflow`가 hidden/auto/scroll인 첫 조상을 찾아** 그걸 제거하거나, 또는 **Portal로 빼서 렌더링**하는 방식으로 대응하면 된다.
