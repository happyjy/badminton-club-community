# 기획서: 병가/휴회 기간 관리 및 회비 정산 반영

**문서 버전**: 1.0  
**대상 기능**: 회원 병가·휴회 기간 등록 및 회비 의무 월 반영

---

## 1. 개요

### 1.1 목적

- 회원의 **병가·휴회 기간**을 등록·관리하고, 해당 기간은 **회비 의무에서 제외**한다.
- 휴회 **횟수·사유**를 기록해 “몇 번 휴회했는지” **평가**할 수 있게 한다.
- **기간 제한 없음**: 출산 등 장기 휴회(예: 3개월 초과)도 허용한다.

### 1.2 범위

| 구분        | 내용                                                              |
| ----------- | ----------------------------------------------------------------- |
| 대상 페이지 | 클럽 회원 상세 (`/clubs/[id]/members/[userId]`)                   |
| 대상 사용자 | 클럽 관리자                                                       |
| 연동 기능   | 회비 대시보드, 미납 조회, 입금 확정, 일괄 확정 제안(안내 대상 월) |

---

## 2. 배경 및 현황

### 2.1 현행

- **회비 입금 시작월**(`feeObligationStartAt`)만 회원 상세에서 설정 가능.
- 회원 상태에 `ON_LEAVE`(휴가)가 있으나 **기간·횟수·사유**는 관리되지 않음.
- “몇 번 휴회했는지” 평가 불가, 휴회 기간이 회비 의무에 반영되지 않음.

### 2.2 요구사항

- **병가/휴회 처리**: 휴회·병가·출산 등으로 쉬는 기간을 별도로 관리.
- **기간 제한 없음**: 3개월 등 상한 두지 않음(출산 등 장기 휴회 허용).
- **몇 번인지 평가**: 휴회 기간을 여러 건 등록해 “1회차, 2회차…” 또는 “총 N회”로 파악 가능.
- **회비 정산 반영**: 휴회 기간에 해당하는 연·월은 회비 의무에서 제외.

---

## 3. 기능 요구사항

### 3.1 휴회 기간 등록·관리

| ID  | 요구사항       | 상세                                                                                    |
| --- | -------------- | --------------------------------------------------------------------------------------- |
| F-1 | 휴회 기간 등록 | 시작 연·월 필수, 종료 연·월·사유 선택 입력. 기간 제한 없음(종료 미입력 시 “미정” 처리). |
| F-2 | 휴회 기간 수정 | 이미 등록한 휴회 건의 시작/종료 연·월, 사유 수정.                                       |
| F-3 | 휴회 기간 삭제 | 등록한 휴회 건 삭제.                                                                    |
| F-4 | 휴회 목록 조회 | 해당 회원의 휴회 기간 목록을 회차(또는 등록일) 순으로 표시.                             |

### 3.2 회비 의무 반영

| ID  | 요구사항                 | 상세                                                                                      |
| --- | ------------------------ | ----------------------------------------------------------------------------------------- |
| F-5 | 의무 월에서 휴회 월 제외 | 특정 연·월이 휴회 구간에 포함되면 해당 월은 회비 의무 없음.                               |
| F-6 | 대시보드 반영            | 회비 대시보드에서 회원별 의무 월·납부 현황 계산 시 휴회 월 제외.                          |
| F-7 | 미납 목록 반영           | 미납 회원 조회 시 해당 월이 휴회인 회원은 미납 목록에서 제외.                             |
| F-8 | 입금 확정 반영           | 입금 확정 시 선택한 연·월이 휴회 구간이면 의무 없음으로 처리(정책에 따라 에러/경고 가능). |
| F-9 | 안내 대상 월 반영        | 일괄 확정 제안(suggestMonths) 등 “안내도 되는 달” 계산 시 휴회 월 제외.                   |

### 3.3 비기능 요구사항

| ID   | 요구사항    | 상세                                                               |
| ---- | ----------- | ------------------------------------------------------------------ |
| NF-1 | 권한        | 휴회 기간 CRUD는 클럽 관리자만 가능(기존 회비 입금 시작월과 동일). |
| NF-2 | 데이터 검증 | 시작 연·월 필수, 종료 입력 시 종료 ≥ 시작, 사유 길이 제한 등.      |

## 4. 데이터 설계

### 4.1 신규 엔티티: 휴회 기간 (MemberLeave)

| 필드         | 타입                  | 필수 | 설명                                    |
| ------------ | --------------------- | ---- | --------------------------------------- |
| id           | Int (PK)              | ○    |                                         |
| clubMemberId | Int (FK → ClubMember) | ○    | 대상 회원                               |
| startYear    | Int                   | ○    | 휴회 시작 연도                          |
| startMonth   | Int                   | ○    | 휴회 시작 월 (1~12)                     |
| endYear      | Int                   | ×    | 휴회 종료 연도 (null = 미정/장기)       |
| endMonth     | Int                   | ×    | 휴회 종료 월 (null = 미정)              |
| reason       | String                | ×    | 사유 (병가, 출산, 일반 휴가 등, 평가용) |
| createdAt    | DateTime              | ○    | 등록일                                  |

- 관계: ClubMember 1 : N MemberLeave.
- 한 회원당 휴회 기간 여러 건 등록 가능 → “몇 회차 휴회” 평가 가능.

### 4.2 기존 로직 확장

- **의무 월 계산**: `feeObligation.ts` 등에서 사용하는 “의무 월” 판단에 **휴회 구간** 인자 추가.
  - 해당 연·월이 어떤 휴회 기간에 포함되면 의무 아님.
  - `endYear`/`endMonth`가 null이면 “해당 월 ≥ 시작 연월”이면 휴회로 간주.

## 5. API 설계

### 5.1 휴회 기간 CRUD

| 메서드 | 경로                                                | 설명                                                                |
| ------ | --------------------------------------------------- | ------------------------------------------------------------------- |
| GET    | `/api/clubs/[id]/members/[userId]/leaves`           | 해당 회원 휴회 기간 목록 조회                                       |
| POST   | `/api/clubs/[id]/members/[userId]/leaves`           | 휴회 1건 등록 (startYear, startMonth, endYear?, endMonth?, reason?) |
| PATCH  | `/api/clubs/[id]/members/[userId]/leaves/[leaveId]` | 휴회 1건 수정                                                       |
| DELETE | `/api/clubs/[id]/members/[userId]/leaves/[leaveId]` | 휴회 1건 삭제                                                       |

- 권한: 클럽 관리자.
- 검증: startYear/startMonth 필수, end 입력 시 end ≥ start, reason 길이 제한 등.

### 5.2 회원 조회 확장 (선택)

- GET `/api/clubs/[id]/members/[userId]` 응답에 `leaves: MemberLeave[]` 포함하거나, 프론트에서 별도 GET leaves 호출로 목록 조회.

## 6. 화면 설계

### 6.1 회원 상세 페이지

- **위치**: `/clubs/[id]/members/[userId]`
- **추가 블록**: “회비 입금 시작월” 아래 **“휴회/병가 기간”** 섹션

**표시 내용**

- 휴회 기간 목록: 시작 연월 ~ 종료 연월(없으면 “미정”), 사유.
- 회차 또는 등록일 순 정렬.

**동작**

- [휴회 추가]: 시작 연월 필수, 종료 연월·사유 선택 입력. (기간 제한 없음.)
- 각 건 [수정] / [삭제].

## 8. 구현 순서

1. **스키마**: `memberLeave.prisma` 추가, `club.prisma`에 relation 추가, `build-schema.ts` 등록, `pnpm run build:schema` 및 마이그레이션.
2. **의무 월 로직**: `feeObligation.ts`에 휴회 구간 타입·헬퍼 및 기존 함수에 leavePeriods 반영, 테스트 보강.
3. **API**: 휴회 CRUD API 및 (선택) 회원 조회에 leaves 포함.
4. **회비 연동**: 대시보드·미납·확정·일괄 확정에서 휴회 데이터 조회 후 의무 월 계산에 반영.
5. **UI**: 회원 상세 페이지에 휴회 섹션(목록/추가/수정/삭제) 구현.

## 9. 정책·참고사항

- **기간 제한 없음**: endYear/endMonth nullable로 “종료 미정” 허용.
- **평가**: 1회원당 여러 MemberLeave 건으로 “N회차 휴회” 및 사유별 구분 가능.
- **회비**: 해당 연·월이 휴회 구간이면 의무에서 제외해 대시보드·미납·확정·안내 대상 월에 일관 반영.
- **기존 “안내도 되는 달”**: 이미 “회비를 언제부터 내야 하는지”(의무 월)와 동일 기준 사용 중. 휴회를 의무 월에서 제외하면 안내 대상 달도 자동으로 휴회가 반영됨.
