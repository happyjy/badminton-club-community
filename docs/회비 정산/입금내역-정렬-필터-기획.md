# 입금내역 처리 – 정렬·필터 기능 구현 기획

## 1. 목표

입금 내역 처리 페이지(`/clubs/[id]/membership-fee/process`)에서 **정렬**과 **필터**를 지원하여, 거래일·입금자명·금액·매칭 회원 기준으로 목록을 조회·정렬할 수 있게 한다.

---

## 2. 정렬 (Sort)

### 2.1 정렬 대상 컬럼

| 컬럼      | 데이터 타입 | 정렬 기준                                |
| --------- | ----------- | ---------------------------------------- |
| 거래일    | Date        | 날짜 오름차순/내림차순                   |
| 입금자명  | string      | 문자열(가나다) 오름차순/내림차순         |
| 금액      | number      | 숫자 오름차순/내림차순                   |
| 매칭 회원 | string      | 표시 문자열(이름 나열) 오름차순/내림차순 |

### 2.2 동작

- **기본 정렬**: 거래일 최신순(내림차순) 권장.
- **1차 정렬만 지원**: 한 번에 하나의 컬럼만 정렬 (복합 정렬 없음).
- **토글**: 같은 컬럼 헤더 클릭 시 오름차순 ↔ 내림차순 전환.
- **처리 위치**: **클라이언트** — 이미 `usePaymentRecords`로 전체 목록을 받아오므로, 받은 `records` 배열을 정렬한 뒤 테이블에 전달.

### 2.3 UI

- 테이블 헤더(`<th>`)를 클릭 가능하게 하고, 현재 정렬 컬럼 + 방향(▲/▼) 아이콘 표시.
- 또는 테이블 위에 "정렬: [거래일 ▼] [입금자명] [금액] [매칭 회원]" 형태의 버튼 그룹.

---

## 3. 필터 (Filter)

### 3.1 필터 항목별 스펙

| 필터 항목 | 타입      | 방식                                               | 비고                                               |
| --------- | --------- | -------------------------------------------------- | -------------------------------------------------- |
| 거래일    | 범위      | 시작일 ~ 종료일                                    | Date picker 또는 input type="date"                 |
| 입금자명  | 범위      | "시작 문자" ~ "끝 문자" (가~하) 또는 **포함 검색** | 범위면 초성/문자열 비교, 포함이면 검색어 일부 매칭 |
| 금액      | 범위      | 최소 금액 ~ 최대 금액 (원)                         | 숫자 입력, 빈 값이면 해당 쪽 제한 없음             |
| 매칭 회원 | 다중 선택 | 선택한 회원이 **포함된** 건만                      | 회원 목록(멀티 셀렉트), 1명이라도 매칭되면 표시    |

- **범위**는 "이상·이하" 또는 "초과·미만" 중 일관된 규칙으로 정의 (권장: 이상·이하).

### 3.2 입금자명 필터 상세

- **옵션 A – 문자 범위**: "시작 문자" ~ "끝 문자" (예: "가" ~ "바") → 해당 구간 이름만 표시.  
  구현: `depositorName.localeCompare(start) >= 0 && depositorName.localeCompare(end) <= 0`.
- **옵션 B – 포함 검색**: 입력한 문자열이 입금자명에 **포함**된 건만 표시.  
  구현: `depositorName.includes(keyword)` (대소문자 무시 선택).
- **권장**: 사용성은 **포함 검색(B)**이 좋고, "범위" 요구사항을 문자 범위(A)로 해석할 수도 있음.  
  기획 단계에서 **포함 검색**으로 정한 뒤, 필요 시 "이름 범위(가~하)"는 추가 옵션으로 확장 가능.

### 3.3 필터 적용 순서

1. 기존 **상태 필터**(전체/대기/매칭됨/확정/에러/건너뜀) — URL `status` 쿼리 유지.
2. 그 다음 **거래일 / 입금자명 / 금액 / 매칭 회원** 필터를 순서 없이 모두 AND 조건으로 적용.

### 3.4 처리 위치

- **클라이언트**: `usePaymentRecords`로 받은 `records`를 상태/필터 조건으로 한 번 더 필터링.
- 서버 API는 그대로 두고, 응답 배열만 클라이언트에서 정렬·필터링.

---

## 4. 상태 관리

| 항목      | 저장 위치           | 이유                                |
| --------- | ------------------- | ----------------------------------- |
| 상태 필터 | URL query           | 이미 구현됨, 공유·북마크 가능       |
| 정렬      | 로컬 state          | 테이블 보기 편의용, URL 복잡도 회피 |
| 필터      | 로컬 state 또는 URL | 공유 필요 시 URL, 아니면 로컬 state |

- **제안**: 정렬·필터 모두 **로컬 state**로 시작.  
  나중에 "현재 필터/정렬 상태 공유"가 필요하면 정렬·필터도 query로 올릴 수 있음.

---

## 5. 구현 단계 제안

### Phase 1 – 정렬

1. **정렬 상태**: `sortBy: 'transactionDate' | 'depositorName' | 'amount' | 'matchedMember'`, `sortOrder: 'asc' | 'desc'`.
2. **정렬 함수**: `PaymentRecord` 배열을 받아 `sortBy`/`sortOrder`에 따라 정렬한 새 배열 반환 (불변).
3. **적용 위치**:
   - **A안** – `process.tsx`에서 `records`를 정렬한 뒤 `PaymentRecordTable`에 전달.
   - **B안** – `PaymentRecordTable`에 `records` + `sortBy`/`sortOrder`를 넘기고, 테이블 내부에서 정렬.  
     **권장**: 한 곳에서만 정렬 로직 보유. **process.tsx**에서 정렬·필터 적용 후 테이블에는 "이미 정렬·필터된 목록"만 넘기는 **A안** 권장 (테이블은 표시만 담당).
4. **UI**: 테이블 헤더 클릭으로 정렬 컬럼/방향 변경. 현재 정렬 컬럼에 ▲/▼ 표시.

### Phase 2 – 필터 UI·상태

1. **필터 상태 타입** (예시):
   ```ts
   type PaymentRecordFilters = {
     transactionDateFrom?: string; // ISO date or YYYY-MM-DD
     transactionDateTo?: string;
     depositorNameKeyword?: string; // 포함 검색 (또는 start/end 문자)
     amountMin?: number;
     amountMax?: number;
     matchedMemberIds?: number[]; // 선택한 회원 ID 배열
   };
   ```
2. **필터 UI**: 테이블 위 영역에 "필터" 접기/펼치기 패널.
   - 거래일: `input type="date"` 2개 (부터 ~ 까지).
   - 입금자명: `input type="text"` (포함 검색) 또는 범위 시 문자 2개 입력.
   - 금액: `input type="number"` 2개 (최소, 최대).
   - 매칭 회원: 기존 `MemberMultiSelectDropdown` 재사용 또는 회원 다중 선택 컴포넌트.
3. "필터 초기화" 버튼 제공.

### Phase 3 – 필터 로직

1. **process.tsx**에서:
   - `usePaymentRecords`로 받은 `records`에 대해
   - `filters` 객체로 필터링 함수 적용 → `filteredRecords`
   - `filteredRecords`를 정렬 → `sortedRecords`
   - `PaymentRecordTable`에는 `records={sortedRecords}` 전달.
2. **필터 함수**:
   - 거래일: `record.transactionDate >= from && record.transactionDate <= to` (날짜만 비교 시 시각 00:00 기준 통일).
   - 입금자명: `depositorName.includes(keyword)` (trim 후, 대소문자 무시 optional).
   - 금액: `(amountMin == null || record.amount >= amountMin) && (amountMax == null || record.amount <= amountMax)`.
   - 매칭 회원: `matchedMemberIds.length === 0` 이면 통과, 아니면 `getRecordMemberIds(record)`와 교집합이 있는지 확인.

### Phase 4 – 소통 문구

- 필터 적용 시: "전체 N건 중 필터 결과 M건" 또는 "N건 (필터 적용: M건)" 형태로 표시.
- 정렬 적용 시: 헤더에 "거래일 ▼" 등으로 현재 정렬만 표시해도 충분.

## 7. 정리

- **정렬**: 거래일, 입금자명, 금액, 매칭 회원 — 1차 정렬, 오름차순/내림차순 토글, **클라이언트** 처리.
- **필터**:
  - 거래일·금액 = **범위**(시작일~종료일, 최소~최대 금액).
  - 입금자명 = **범위(가~하)** 또는 **포함 검색** 중 선택 (권장: 포함 검색).
  - 매칭 회원 = **다중 선택** (선택한 회원이 포함된 건만).
- **구현 위치**: 정렬·필터 모두 클라이언트. `process.tsx`에서 records → 필터 → 정렬 → 테이블 전달.
- **상태**: 정렬·필터는 로컬 state로 시작, 필요 시 URL query로 확장 가능.

이 기획대로 적용하면 입금내역 처리 화면에서 정렬과 범위/다중선택 필터를 일관되게 사용할 수 있습니다.

## 8. 회비 관리 대시보드 – 회원별 납부 현황 필터

> **페이지**: `/clubs/[id]/membership-fee` (`index.tsx`)  
> **맥락**: 입금 확정으로 쌓인 `MembershipPayment`를 연도·회원별로 집계한 테이블. 입금내역 **처리**와는 화면이 다르지만, 미납 추적·독촉 목록을 위해 같은 문서 묶음에서 다룬다.

### 8.1 목표

- 전체 회원 중 **미납이 있는 사람만** 또는 **특정 기간까지 납부를 마친 사람만** 빠르게 본다.
- 입금내역 처리 화면의 필터(거래일·입금자명 등)와 **역할이 다름**: 여기서는 **회원 × 월 납부 여부** 기준.

### 8.2 UI (라디오 미사용)

| 모드      | UI                                                  |
| --------- | --------------------------------------------------- |
| 전체      | 세그먼트 버튼 중 하나 선택                          |
| 미납 있음 | 동일                                                |
| 납부 완료 | 동일 + **기준 마지막 달** `<select>` (1월~12월까지) |

- 세그먼트: 회색 배경 + 선택 탭만 흰색 강조(토글 그룹 형태).

### 8.3 필터 규칙

| 모드          | 대상 회원      | 조건                                                                                                    |
| ------------- | -------------- | ------------------------------------------------------------------------------------------------------- |
| **전체**      | 면제 포함 전원 | 필터 없음                                                                                               |
| **미납 있음** | 일반·부부만    | 선택 연도 **1월~12월** 중 **한 달이라도** 미납(`payments[month] === false`)이면 포함. 12개월 전부 확인. |
| **납부 완료** | 일반·부부만    | **1월 ~ N월**까지 **모두** 납부한 회원만. N은 드롭다운(1~12). N+1~12월 납부 여부는 보지 않음.           |
| 면제          | —              | 미납/납부완료 필터에서는 **제외**(납부 의무 없음). 전체 목록에서만 표시.                                |

### 8.4 구현 위치

| 파일                        | 내용                                                                                                                                  |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| `membership-fee/index.tsx`  | `memberFilter` state(`all` \| `unpaid` \| `paid`), `paidThroughMonth`, 세그먼트 + select, 목록 필터 후 `PaymentDashboardTable`에 전달 |
| `PaymentDashboardTable.tsx` | `memberHasAnyUnpaidMonthInYear`, `memberFullyPaidThroughMonth` export. 테이블 셀의 빨간 X는 기존처럼 **지난 달·이번 달** 기준(표시용) |
