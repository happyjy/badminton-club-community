## 2. 구현 완료된 기능

- 엑셀 업로드 → 회원 매칭(이름 포함, 성 생략, 부부 2명 등)
- 가입비(FeeType "가입비") 미매칭 처리
- 회비 유형 관리(FeeType CRUD), 월·연납만(분기·반기 주석 처리)
- 확정 시 연도 선택(전년/당년/내년)
- 월 선택: 단일 / Ctrl+토글 / Shift+범위 선택·해제

## 3. 핵심 플로우

1. **엑셀 업로드** → PaymentRecord 생성, `matchedMembers`(다중 회원) 저장
2. **검토·수정** → 회원 매칭 수정, 매칭 실패 시 건너뛰기
3. **확정** → 선택 연도·월로 MembershipPayment 생성, 상태 CONFIRMED
4. **일괄 확정** → 매칭된 건만 선택 후 일괄 확정

## 4. 주요 데이터 모델

- **PaymentRecord**: batchId, depositorName, amount, matchedMemberId, status, matchedMembers(다중)
- **PaymentRecordMatchedMember**: paymentRecordId, clubMemberId(다중 매칭)
- **MembershipPayment**: clubMemberId, year, month, amount, paymentRecordId
- **MatchResult**: memberId, memberIds?(부부 시 2명)

## 5. 추가 작업 시 참고

- `handleConfirm(recordId, year, months)` 시그니처
- confirm API: matchedMembers 우선, 없으면 matchedMemberId 사용
- 부부 2명 매칭(동일 부부 그룹 전원): 월 합계는 coupleAmount 1회(세대당); 일괄 확정 시 MembershipPayment는 합계 coupleAmount가 되도록 인원 수로 균등 분배

## 6. 입금 부족/초과(부분 납부) 처리 (옵션 A)

- **부족**: 커플 2명 50,000원 예상인데 40,000원만 입금 후 나중에 10,000원 추가 납부
- **초과**: 월회비 25,000원 + 가입비 100,000원 = 125,000원 한 건 입금 시, 1개월 월회비만 인정
- **업로드**: 금액이 1개월 예상액 미만이어도 회원 매칭되면 MATCHED 저장
- **확정**: 입금액 < 예상액 → 부족 확정(금액 균등 배분) / 입금액 >= 예상액 → 선택한 월·인원만큼만 확정(초과분 미할당)
- **일괄 확정**: 부족 건은 실패 처리, "개별 확정에서 월 선택 후 부족 확정해주세요" 안내

### 구현 요약

| 구분          | 파일              | 변경 내용                                                                                          |
| ------------- | ----------------- | -------------------------------------------------------------------------------------------------- |
| 업로드 API    | `upload.ts`       | 회원 매칭됐는데 금액 검증 실패 시, 1개월 예상액 미만이면 `MATCHED` 저장                            |
| 확정 API      | `confirm.ts`      | 입금액 >= 예상액 → 선택한 월만 확정(초과분 미할당) / 입금액 < 예상액 → 부족 확정(금액 균등 배분) |
| 일괄 확정 API | `bulk-confirm.ts` | 부족 건 실패, 사유: "입금 부족 - 개별 확정에서 월 선택 후 부족 확정해주세요"                       |

**사용 흐름 예시 (커플 2명, 50,000원 예상, 40,000원 입금)**  
1. 엑셀 업로드 → 회원 매칭 시 `MATCHED`로 저장  
2. 입금 내역 처리 페이지에서 해당 건 선택  
3. 연도·월 선택 후 확정 → 2명 각 20,000원으로 확정  
4. 나중 10,000원 추가 입금 시 별도 PaymentRecord로 업로드 후 확정  

**사용 흐름 예시 (1명, 월회비 25,000 + 가입비 100,000 = 125,000원 한 건)**  
1. 엑셀 업로드 → 회원 매칭 시 `MATCHED`로 저장 (125,000 = 25,000×5이므로 5개월 제안될 수 있음)  
2. 1개월만 선택 후 확정 → 25,000원 1개월 회비로 확정, 초과분(가입비 100,000원)은 미할당  
3. 가입비는 별도 정산 또는 현장에서 처리
